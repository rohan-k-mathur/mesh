## Vocabulary

Work in the **quotient ECC** `ECC/≈` (D1 §0): arrows identified by endpoints, per-derivation assumption sets, and derivation multiplicity, **modulo derivation-ID renaming** (the L1.2 finding — `transport`/`compose` re-associate composite derivation IDs, which `≈` quotients). In `ECC/≈`, `compose` is strictly associative and unital.

The **Plexus bicategory 𝓟** (D1 §1): 0-cells = rooms (ECCs over [`lib/argumentation/ecc.ts`](../../lib/argumentation/ecc.ts)); 1-cells = transport functors (`Functor` partial claim-map + the strict arrow action `transport(F, ·)`); 2-cells `α: F ⇒ F'` = families `(α_c: F(c) → F'(c))_c` of room-`B` arrows natural in source arrows; vertical/horizontal composition, whiskering, identities as in D1 §1.4.

## Supporting lemmas (folded discharges)

## Statement

Let `(·)_∗` denote transport from 1-cells of 𝓟 to lax functors on `ECC/≈`. Define the **round-trip 2-cell** `η` of a directed cycle of transport functors `W: A → A` by `η_c: c → W(c)` (the claimMap composite walked around the cycle, per claim). Define **𝓟°** as the full sub-bicategory spanned by the rooms and transport functors all of whose directed cycles have **invertible** `η` (each `η_c` an ECC iso — equivalently, the claim returns to itself *up to ECC inter-derivability*).

> **T010 (Plexus coherence, symbolic).**
> 1. **(Bicategory + lax functor.)** 𝓟 is a bicategory and `(·)_∗` is a lax functor into it; its comparison 2-cells `γ_{G,F}: G_∗∘F_∗ ⇒ (G∘F)_∗` are the **identity** on the total part (claims with images under both composites) and **undefined** on drops. The pentagon and triangle hold.
> 2. **(Pseudofunctor ⟺ monodromy-free.)** `(·)_∗` restricts to a **pseudofunctor** on a region (every `γ` present and invertible) **iff** that region is **𝓟°** (every cycle's `η` invertible). **𝓟° is the largest** such sub-bicategory.
> 3. **(Faithfulness boundary.)** T010(1–2) is a theorem about **symbolic** transport. It is faithful to the live materialized pipeline ([`app/api/room-functor/apply/route.ts`](../../app/api/room-functor/apply/route.ts)) **exactly on the strict materialization path** (`depth>1` and non-empty `claimMap`, on the new-import branch), where premise structure is preserved; on the **lax** path it over-claims (a symbolically iso-closed cycle can be materially lossy). The boundary is **decidable** from the `apply` call.

## Proof (map)

- **(1)** D1 establishes 𝓟's data and W1–W3. D2 §3.1 computes `γ`: since object composition is the on-the-nose partial-map composite (L1.5 / A1) and `transport` is strict (L1.1), `G(F(c)) = (G∘F)(c)` definitionally, so `γ` is the identity on the total part and absent on drops; D2 §3.2–§3.3 discharge the pentagon and triangle, which collapse (both sides = the identity 2-cell on the common total part; total parts coincide by A1 associativity) — the manipulation is legalized by W2. ∎(1)
- **(2)** D3 §2 bridges `γ` to `η` componentwise (γ present+invertible ⟺ η invertible), using D2's `γ`-shape. D3 §3 proves the biconditional `isPseudofunctor ≡ isMonodromyFreeIso` over the four fates (`closed`, `drift-iso`, `drift-noniso`, `dropped`), with the (⇒) direction the B2b identification (a `dropped`/`drift-noniso` claim ⟹ a missing/non-invertible `γ`); maximality is the contrapositive. The **iso-closure** definition (not strict-identity) is forced by a `drift` between inter-derivable claims, which is an invertible 2-cell (D3 §4). ∎(2)
- **(3)** D4 reads the two materialization paths in `apply/route.ts`: the strict path (`reconstructArgumentStructure` + `recursivelyImportPremises`) preserves the assumptions that witness the `drift-iso`, so materialized 2-cells equal the symbolic `γ`; the lax (text-only) path strips them, breaking invertibility. The boundary predicate is `branch="new" ∧ depth>1 ∧ |claimMap|>0`. ∎(3)

## What this establishes

- The coherence half of the Direction-4 program: transport composes soundly (is a pseudofunctor) **exactly** on the monodromy-free-up-to-iso region, and the one-hop production contract is the correct conservative default off it. This is the prerequisite the settled "coherence first, sheaf-cohomology after" sequencing names — the quantitative $H^1(\mathbb{R})$ (sub-program B) is only well-posed on 𝓟° (holonomy needs transport to compose).
- **Live finding:** the faithful region is the **existing** strict materialization path, not a greenfield build — so the gated production change is "make strict the default," narrower than the A0 audit implied.

## What this rules out (for the implementation)

- **No multi-hop transport on the lax path.** Until the gated PRs land (L2 origin-dedupe band + strict materialization as default), `transport` must stay one-hop in production; the symbolic theorem does not license multi-hop where the band double-counts or premises are dropped.
- **No claim-id-equality coherence checks.** Any "is this cycle coherent?" surface must test **ECC inter-derivability** (iso-closure), not claim-id equality — else it mis-flags `drift-iso` as an obstruction (the D3 §7 / H2 refinement).
- **No quantitative-class claims from this theorem.** T010 certifies *qualitative* soundness only; a non-trivial $H^1(\mathbb{R})$ remains gated on an independent per-edge transport weight (Q-042 offshoot, open).

**Non-blocking clarifications (author follow-up; none gate the sign-off).**

1. **The live probe's inter-derivability oracle is a *necessary-but-not-sufficient* proxy for the symbolic ECC-iso.** `holonomy`'s `interDerivable = reach(r,a,b) ∧ reach(r,b,a)` is two-way *premise→conclusion reachability* and **ignores assumptions**, whereas the symbolic 𝓟° predicate `isIsoVia` requires the round-trip to compose to an *assumption-free* identity. So the probe's `drift-iso` bucket can **over-count** 𝓟° when the two-way derivations carry net assumptions. The *theorem* is unaffected (T010(2) is defined and tested with the strict `isIsoVia`); this is an instrument-fidelity caveat that should ride alongside the D4 "probe verdict is symbolic only" annotation — ideally the oracle also checks assumption cancellation before counting a landing as `drift-iso`.
2. **D4 row-name imprecision.** D4's table says the strict path "preserves / lax drops `ArgumentPremise` rows." In the route the strict path (`reconstructArgumentStructure`) writes diagram-level `InferencePremise` rows and recursively imports premise *Arguments* (when `claimMap` maps them); it does not itself write `ArgumentPremise` rows, while the probe's reachability oracle *reads* `ArgumentPremise`. The qualitative boundary (strict preserves Toulmin/premise structure; lax is text-only) and the decidable predicate are correct; only the row name in the prose is loose — read "premise/Toulmin structure (diagram `InferencePremise` + recursively-imported premise arguments)."
3. **Test signature is complete only for single-derivation witnesses.** `sigArrow` collapses the per-derivation assumption *multiset* to a flat union, so it is a complete `≈`-invariant only on single-derivation arrows (all D1–D3 corroboration uses such witnesses). The underlying laws (W1–W3, pentagon) hold for *multi-derivation* 2-cells too — `compose`/`join` genuinely associate and interchange on derivation multisets, with per-triple assumption unions invariant under reassociation — so this is a test-coverage caveat, not a soundness gap; worth stating that multi-derivation 2-cells are covered by the proof but not the suite.
