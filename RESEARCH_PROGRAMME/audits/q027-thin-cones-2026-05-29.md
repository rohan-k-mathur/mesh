# Q-027 thin-cones diagnostic — aspirin worked example

**Status:** complete (steps 1–5)
**Started:** 2026-05-29
**Completed:** 2026-05-29
**Gates:** C001b (b₁ surjectivity)
**Method:** worked example, paper diagnostic, no mechanisation
**Source instance:** Ambler 1996 §1.1 Example 1 (medical diagnosis / aspirin)
**Headline verdict:** Q-027 closes **negatively in Reading A under the per-cone bridge target** (safe and compressed encodings agree). However, the diagnostic surfaced an alternative bridge target — `𝒫_fin(Inc(B))` instead of per-cone `Cᵢ` — that yields a clean positive answer trivially. Net effect: C001b's current framing needs revision, not abandonment. See §7.

## 0. Why this instance instead of synthetic

Pre-flight (2026-05-29) confirmed that the substrate's Phase 2e/2f artefacts
provide the structural framing (cone decomposition, Daimon Lock) but no concrete
behaviour. Ambler's own primary worked example, by contrast, gives a fully
specified Ambler-side hom-semilattice presentation: `𝒮/Γ(A, B) = 𝒫_fin(𝒞/Γ(A, B))`
under set union, where `𝒞/Γ` is the simply-typed λ-calculus over Γ. Reusing
Ambler's Example 1 means the Ambler-side enumeration is mechanical and the
Q-027 effort concentrates on the Ludics-side encoding and the bridge map —
which is where the open question actually lives.

A secondary benefit: a completed worked example here will also discharge one
of C001b's positive-settlement requirements (a worked instance on a non-trivial
behaviour), so this diagnostic doubles as substrate work.

## 1. The context Γ (Ambler Example 1)

Five labelled rules, each with a confidence value `θᵢ ∈ [0,1]` (irrelevant to
the underlying-category enumeration; confidence lives in `C_semi` and is
erased by `U`):

| Label | Type                                                    |
|-------|---------------------------------------------------------|
| t₁    | `muscle_pain → aspirin`                                 |
| t₂    | `(muscle_pain ∧ ¬gastric_ulcer) → aspirin`              |
| c₁    | `gastric_ulcer → ¬aspirin`                              |
| i₁    | `stomach_pain → gastric_ulcer`                          |
| i₂    | `(short_term ∧ anxiety) → ¬gastric_ulcer`               |

Atomic propositions: `muscle_pain`, `stomach_pain`, `short_term`, `anxiety`,
`aspirin`, `gastric_ulcer`, `¬aspirin`, `¬gastric_ulcer`.

Negation is treated as a primitive atom in Ambler's setting (no classical
involution available — consistent with the sub-classical / argumentation
reading and with the Ludics polarity discipline).

The base category `𝒞/Γ` is the free simply-typed λ-calculus over Γ.

## 2. Ambler-side enumeration: `𝒞/Γ(A, B)` for three queries

For each query `A ⇒ B`, we enumerate the β-normal η-long λ-terms of type `B`
in the context `(x : A, Γ)`. Equivalently: the inhabitants `𝒞/Γ(A, B)`
returned by Ambler's ATP modulo the βη-normalisation it implicitly performs.

We use the convention `fst`, `snd` for product projections and write
application left-associatively. We compare to the ATP transcripts in
[AMBLER_PAPER.md](../../Development%20and%20Ideation%20Documents/ARCHITECTURE/AMBLER_PAPER.md)
lines 200–220.

### 2.1 Query Q1: `muscle_pain ∧ stomach_pain ⇒ aspirin`

Variables in context `(x : mp ∧ sp, Γ)`:

- `fst(x) : muscle_pain`
- `snd(x) : stomach_pain`

To produce `aspirin`, only `t₁` and `t₂` apply. Case analysis:

- **Via `t₁` (needs `muscle_pain`):** `t₁ fst(x) : aspirin`. ✓
- **Via `t₂` (needs `muscle_pain ∧ ¬gastric_ulcer`):** Need `¬gastric_ulcer`.
  Only producer is `i₂`, which needs `short_term ∧ anxiety`. Not derivable
  from the context. **No term.**

**Enumeration result.**

$$\mathcal{C}/\Gamma(Q_1.A,\ \text{aspirin}) \;=\; \{\, t_1\, fst(x) \,\}$$

Cardinality: **1.** Matches ATP `[t₁ fst(0)]`.

### 2.2 Query Q2: `muscle_pain ∧ stomach_pain ⇒ ¬aspirin`

Same context as Q1. Only `c₁` produces `¬aspirin`, needing `gastric_ulcer`.
Only `i₁` produces `gastric_ulcer`, needing `stomach_pain = snd(x)`.

- **Via `c₁ ∘ i₁`:** `c₁ (i₁ snd(x)) : ¬aspirin`. ✓

**Enumeration result.**

$$\mathcal{C}/\Gamma(Q_2.A,\ \neg\text{aspirin}) \;=\; \{\, c_1\, (i_1\, snd(x)) \,\}$$

Cardinality: **1.** Matches ATP `[c₁ (i₁ snd(0)))]`.

### 2.3 Query Q3: `muscle_pain ∧ stomach_pain ∧ short_term ∧ anxiety ⇒ aspirin`

Context `(x : mp ∧ (sp ∧ (st ∧ an)), Γ)` (right-associative ∧). Variables:

- `fst(x) : muscle_pain`
- `fst(snd(x)) : stomach_pain`
- `fst(snd(snd(x))) : short_term`
- `snd(snd(snd(x))) : anxiety`

To produce `aspirin`:

- **Via `t₁`:** `t₁ fst(x) : aspirin`. ✓ (same shape as Q1)
- **Via `t₂` (needs `muscle_pain ∧ ¬gastric_ulcer`):** Now derivable. Need
  `short_term ∧ anxiety = ⟨fst(snd(snd(x))), snd(snd(snd(x)))⟩`. Then
  `i₂ ⟨…⟩ : ¬gastric_ulcer`. Pair with `fst(x)` and apply `t₂`:

  `t₂ ⟨fst(x), i₂ ⟨fst(snd(snd(x))), snd(snd(snd(x)))⟩⟩ : aspirin`. ✓

**Enumeration result.**

$$\mathcal{C}/\Gamma(Q_3.A,\ \text{aspirin}) \;=\; \{\, t_1\, fst(x),\ \ t_2\, \langle fst(x),\, i_2\, \langle fst(snd(snd(x))),\, snd(snd(snd(x)))\rangle\rangle \,\}$$

Cardinality: **2.** Matches ATP `[t₁ fst(0), t₂ pair(fst(0), i₂ snd(snd(0)))]`
modulo the ATP's projection-shorthand for nested products.

### 2.4 Ambler-side hom-semilattice (computed)

By definition of `𝒮/Γ`, the hom-semilattice for each query is the finite
power set of the above generator set, ordered by inclusion, with `∨ = ∪` and
`⊥ = ∅`:

| Query | `|𝒞/Γ(A,B)|` | `|𝒮/Γ(A,B)|` | Structure                              |
|-------|--------------|--------------|----------------------------------------|
| Q1    | 1            | 2            | `{∅, {t₁ fst(x)}}` — 2-chain           |
| Q2    | 1            | 2            | `{∅, {c₁(i₁ snd(x))}}` — 2-chain       |
| Q3    | 2            | 4            | `𝒫({t₁ fst(x), t₂ ⟨…⟩})` — Boolean    |

Q3 is the first non-trivial case (rank-2 free JSL on 2 generators), and is
the case worth driving the Ludics-side comparison against.

### 2.5 Generators that distinguish Q3 from Q1

The two Q3 generators differ by **which set of context atoms they consume**:
`t₁ fst(x)` uses only the `muscle_pain` projection; `t₂ ⟨fst(x), i₂ ⟨…⟩⟩`
uses all four. They are **incomparable** under any reasonable "evidence
support" ordering on λ-terms — neither subsumes the other's atom-usage.

This matters for the bridge: any candidate Ludics encoding must produce a
cone whose generators distinguish exactly these two argument-shapes. If the
Ludics encoding of Q3 yields only one cone-generator (or yields three or
more), the surjectivity question becomes immediately diagnostic.

## 3. Step-3 setup: what the Ludics encoding must produce (preview only)

*Not executed in this round; recorded so step 3 starts from a precise target.*

For Q3 (the load-bearing case), the Ludics-side construction must yield a
behaviour `B_{Q3}` whose incarnations correspond to the answer-paths and whose
cones each match one of the two Ambler generators. Specifically, the test
will be:

> Does there exist a cone `Cᵢ ⊆ B_{Q3}` whose chronicle-extensions of `Dᵢ`
> (i.e. additions of negative branches past daimons) are in bijection with
> the singletons `{t₁ fst(x)}` and `{t₂ ⟨…⟩}` respectively? Or do the two
> Ambler generators force *two distinct cones* `C₁`, `C₂` of `B_{Q3}` (one
> per generator), with each cone thin (i.e. its JSL has only `{Dᵢ}` and
> trivial extensions)?

The second possibility — two thin cones, one generator each — would be the
"thin-cones gate is real" failure: the cone JSL contributes nothing beyond
the singleton, and all the Ambler-side join structure has to come from
*outside* the per-cone JSL (e.g. from a join over `Inc(B)` itself, which
Phase 2e ruled out as a JSL operation). That would be a negative answer to
Q-027 in its current Reading-A form.

The first possibility — one cone whose chronicle-extensions reach both
generators — would be the positive answer: the cone's literal-`∪` extension
fully recovers Ambler's `∨`.

A third possibility — partial coverage (cone reaches some but not all
generators) — would localise an obstruction and may indicate that `⊑` is
needed.

## 4. Provisional verdict (after steps 1–2)

The Ambler-side enumeration is **clean and small**: Q1 and Q2 have 1
generator each (trivial), Q3 has 2 (the first non-trivial test). The diagnostic
is feasible: the Q3 hom-semilattice has 4 elements and 2 generators, both
explicit and incomparable.

**No structural obstruction has surfaced yet on the Ambler side.** The two
Q3 generators are precisely the kind of "different consumption pattern over
the same context" objects we expect Ludics negative-branch extensions to
witness — but whether they do is exactly the open question.

Steps 3–5 remain.

## 5. Step 3 — safe Ludics encoding of Q3

### 5.1 Encoding choices (safe / least-rigged)

- One positive locus per atomic proposition: `ξ_aspirin`, `ξ_mp`, `ξ_sp`,
  `ξ_gu`, `ξ_¬gu`, `ξ_st`, `ξ_an`, `ξ_¬asp`.
- Each rule = one positive ramification at its conclusion locus. So
  `ξ_aspirin` has two ramifications (one focused on `t₁`'s premise `mp`,
  one focused on `t₂`'s premise `mp ∧ ¬gu`); `ξ_gu` has one ramification
  (via `i₁`'s premise `sp`); `ξ_¬gu` has one (via `i₂`'s premise
  `st ∧ an`); `ξ_¬asp` has one (via `c₁`'s premise `gu`).
- Context atoms (`mp`, `sp`, `st`, `an`) are discharged by daimon at the
  appropriate negative leaf.
- The behaviour `B_{Q3}` is the `⊥⊥`-closure of the set of closed
  argument-designs at `ξ_aspirin` over the rules in Γ.

### 5.2 Enumerating `Inc(B_{Q3})`

A design at `ξ_aspirin` begins with a single positive action selecting a
ramification (= selecting which rule it invokes). Two choices: `t₁` or
`t₂`. The smallest design making each choice:

- **`D_{t_1}`**: select `t₁`; negative dispatch on its single premise
  `mp`; positive action selects the context projection providing `mp`
  (= `fst(x)`); daimon.
- **`D_{t_2}`**: select `t₂`; negative dispatch on `mp ∧ ¬gu`; positive
  action selects the pair construction; recursively `D_{mp}` via `fst(x)`
  ending in daimon, and `D_{¬gu}` via `i₂` on `⟨st, an⟩` ending in
  daimon.

Neither is a sub-design of the other — they differ in their **initial
positive action at `ξ_aspirin`** — so by the antichain property of
`Inc(B)`, both are minimal:

$$\mathsf{Inc}(B_{Q3}) \;=\; \{\, D_{t_1},\ D_{t_2}\,\}, \qquad |\mathsf{Inc}(B_{Q3})| = 2.$$

### 5.3 Cone size (the load-bearing observation)

By the Daimon Lock Lemma (Phase 2f §0 prerequisite), designs in the same
cone share their entire positive skeleton and differ **only in
negative-branch coverage**. In the safe encoding:

- At every negative locus encountered in `D_{t_1}` or `D_{t_2}`, there is
  a **unique opponent move to respond to** — atoms have a single producer
  in scope (the appropriate context projection or the appropriate rule).
- Therefore there is no negative branch the design could *elect* to cover
  but currently isn't. No room to grow `D_{t_i}` within `B_{Q3}`.

$$C_{t_1} \;=\; \{D_{t_1}\}, \qquad C_{t_2} \;=\; \{D_{t_2}\}.$$

**Both cones are thin** (singletons). The per-cone JSL on each is the
trivial 1-element JSL.

### 5.4 Bridge check (Reading A, per-cone target — current C001b framing)

The counit `ε : 𝓕(Inc(B_{Q3})) → 𝒮/Γ(Q3.A, aspirin)` sends a formal join
of cone-elements to the actual Ambler join of their λ-term correspondents.
With both cones thin and each contributing only its own bottom
(itself = its incarnation), the image of `ε` is at most:

$$\mathrm{im}(\varepsilon) \;\subseteq\; \{\, \emptyset,\ \{t_1\, fst(x)\},\ \{t_2\,\langle\dots\rangle\}\,\}.$$

The Ambler-side `𝒮/Γ(Q3.A, aspirin)` has **4 elements** (§2.4). The image
has **3**. The missing element is exactly the cross-cone join
`{t_1\, fst(x),\ t_2\,\langle\dots\rangle\}`, which Phase 2e's Cross-Cone
Incompatibility theorem forbids at the design level.

**`ε` is not surjective** under the safe encoding with the per-cone
bridge target.

## 6. Step 3′ — compressed Ludics encoding of Q3

### 6.1 Compression rules attempted

- Rules with the same conclusion share their conclusion locus.
  **No-op:** already the case in the safe encoding — `t₁` and `t₂` both
  ramify at `ξ_aspirin`.
- Atomic propositions appearing in multiple producers conflated.
  **No-op for Q3:** no atom in this fact-base has multiple producers in
  scope.

### 6.2 The genuine compression option: invert polarity at `ξ_aspirin`

The one transformation that *could* let a single design cover both rule
choices is to make `ξ_aspirin` **negative**: opponent challenges, design
dispatches by cases over rule-choices. A single design could then cover
both opponent moves "challenge-via-`t₁`" and "challenge-via-`t₂`" in one
chronicle-set, packing both Ambler generators into one cone.

**This encoding fails the faithfulness test.** Inverting the polarity of
`aspirin` makes it something the **opponent** asserts and the **design**
defends against, which is the opposite of "argument *for* aspirin."
Ambler's `f ∈ 𝒮/Γ(A, B)` is a positive offering by the prover; it must
map to a positive design at the conclusion. The polarity inversion
destroys the bridge it was supposed to enable.

### 6.3 Compressed encoding verdict

No compression that preserves the faithfulness of "argument for B"
changes the picture. **Two thin cones, image of `ε` misses the join,
bridge fails.** Same negative verdict as the safe encoding.

## 7. Step 4 — alternative bridge target surfaced by the analysis

### 7.1 The level mismatch

The enumeration revealed a striking shape match that the per-cone framing
obscures:

| Ambler                                | Ludics (alternative target)              |
|---------------------------------------|------------------------------------------|
| Base set `𝒞/Γ(A,B)` (2 λ-terms for Q3) | Base set `Inc(B)` (2 incarnations for Q3) |
| `𝒮/Γ(A,B) = 𝒫_fin(𝒞/Γ(A,B))`         | `𝒫_fin(Inc(B))` — free JSL on incarnations |
| Join: set union                       | Join: set union                          |
| Bottom: `∅`                            | Bottom: `∅`                               |
| Generator: a single λ-term            | Generator: a single incarnation          |

Under this re-target, `ε : 𝓕(Inc(B)) → 𝒮/Γ(A,B)` is the free extension
of a generator-level bijection `Inc(B_{Q3}) ↔ 𝒞/Γ(Q3.A, aspirin)`
(sending `D_{t_1} ↦ t_1\, fst(x)`, `D_{t_2} ↦ t_2\,\langle\dots\rangle`).
It is then **automatically a JSL isomorphism**. Both b₁ (surjectivity)
and b₂ (injectivity) would close simultaneously and trivially.

### 7.2 Why this is consistent with Phase 2e

The alternative target `𝒫_fin(Inc(B))` is **not a substructure of `B`** —
its elements are *sets of designs*, not designs. So Phase 2e's Cross-Cone
Incompatibility theorem (no design in `B` straddles two cones at the
design level) is **respected**: the set-union operation lives one level
up, where there is no analogous obstruction.

This is essentially the Phase 2e §5.4 "interpretation (b)" reading of
`(D_1 \cup D_2)^{\perp\perp}` — the smallest behaviour containing both —
that Phase 2f §1.2 set aside in favour of interpretation (a) for the
per-cone JSL purpose. Interpretation (b) was the wrong target for *that*
question (per-cone JSL of designs), but it is the **right target for the
Ambler bridge**, because Ambler's `𝒮/Γ` is itself a power-set
construction.

### 7.3 What this does to C001b

C001b's current statement targets the per-cone JSL. Under the safe and
compressed encodings of Q3, **C001b-as-currently-stated fails** (b₁ is
not surjective onto the Ambler hom-semilattice, because the cones are
too sparse).

C001b admits a **natural restatement** at the `𝒫_fin(Inc(B))` level
where b₁ and b₂ become trivial corollaries of a generator-level
bijection. b₃ (erasure-square commutativity) and the bijection itself
become the actual mathematical content.

This is a substantive structural revision, not a refutation — and it is
the kind of finding the small-cone diagnostic was meant to produce.

## 8. Step 5 — resolution and recommended next actions

### 8.1 Resolution of Q-027

**Q-027 resolves NEGATIVELY in Reading A under the per-cone bridge
target**, on the basis of the Q3 worked example under both safe and
compressed encodings. The cones are thin (singletons), the per-cone JSL
is trivial, and the Ambler-side join structure (Boolean-`2` lattice on
2 generators) cannot be recovered as the image of `ε`.

**The negative resolution does not damage the substrate** — it localises
the failure to the choice of bridge target. The per-cone reframe is
correct *as a fact about `B`* (each cone is a JSL by T001) but is the
**wrong place** to seat the bridge into Ambler's hom-semilattice.

### 8.2 Recommended next actions

1. **Update Q-027 in the registry** to status `resolved-negative-with-redirect`, with a one-line summary and pointer to this audit.
2. **Open a new conjecture C001b′** at the `𝒫_fin(Inc(B))` bridge target, stating: `ε : 𝓕(Inc(B)) → 𝒮/Γ(A, B)` is a JSL iso, decomposed as (b₁′) generator-level surjection `Inc(B) → 𝒞/Γ(A, B)`, (b₂′) generator-level injection, (b₃′) erasure-square naturality. The current C001b file should be marked `superseded-by-C001b′` rather than rewritten — its per-cone framing was a faithful attempt at one reading of the substrate and is worth preserving as a record.
3. **File a follow-up question Q-028** on whether the generator-level bijection `Inc(B) ↔ 𝒞/Γ(A, B)` is canonical or depends on encoding choices. The aspirin Q3 example gives bijection on a 2-element set, which is suggestive but not diagnostic (any bijection works). Need a case with `|Inc(B)| ≥ 3` to test canonicality.
4. **Defer Reading-B (`⊑` approximation order) consideration.** Reading A under the new bridge target works. Reading B was the contingency if Reading A failed *everywhere*; that's not what happened.

### 8.3 What this does NOT settle

- **The Q1/Q2 cases were not load-bearing** (1 generator each), so they didn't exercise the join structure. The negative verdict for the per-cone target relies on Q3 alone. A single counterexample suffices for non-surjectivity, but the positive prospects for the re-targeted C001b′ rest on the bijection holding for *all* behaviours, not just Q3. Q-028 is the right place to test that.
- **The erasure-square (b₃)** was not touched. It is a naturality condition between `U` and the bridge; orthogonal to the surjectivity question Q-027 settled.
- **Higher-order Ambler hom-sets** (where `A → B` itself appears as a sub-formula) were not exercised. Ambler's `𝒮/Γ` is defined uniformly for all types, but the Q3 example is propositional. A function-type test case should be on the C001b′ verification list.
