# Hub Overhaul — Dev Roadmap

**Status:** proposed · **Owner:** Rohan · **Created:** 2026-06-30

Overhaul of the deliberation Hub ([app/hub/ui/Hub.tsx](../app/hub/ui/Hub.tsx)) from a
bare list into a real **discovery + sorting** surface, driven by deliberation
**titles**, scoped to **listable (public + viewer's own)** deliberations, and
re-skinned to cohere with the platform design system.

---

## 1. Motivation / current state

The Hub was built early and is now the weakest discovery surface in the product.

**`app/hub/ui/Hub.tsx`** — a client component with a search box, an "any / open
calls" `<select>`, a stubbed tag picker (`TODO`), and a 2-column grid of
unstyled `border rounded` cards. Cards show `title ?? host.type`, dot-joined
tags, an optional call line, and `claims` / `openCQs` stats. Three links per
card: `/deliberation/{id}`, `?mode=panel`, `?mode=synthesis`.

**`app/api/hub/deliberations/route.ts`** — the data source and the root of most
gaps:

- **Hardcodes `title: null` and `tags: []`** (route lines 57–58) even though both
  columns exist and are populated.
- **Search (`q`) only matches `id` and `hostId`** — never the human-readable title.
- **Stats are N+1**: a `claim.findMany` + `cQStatus.count` *per row* (up to 40
  rows → up to 81 queries).
- Uses `listableDeliberationWhere(userId)` — correct, we keep it.

**What already exists (no new infra needed):**

- `Deliberation.title`, `titleAutoGenerated`, `titleUpdatedAt`, `tags String[]`,
  `visibility` enum, `hostType`, and the `calls DeliberationCall[]` relation —
  [schema.prisma:4695](../lib/models/schema.prisma#L4695).
- Titles are auto-populated by
  [lib/deliberations/autoNameEnrichment.ts](../lib/deliberations/autoNameEnrichment.ts)
  (LLM backfill over `{ titleAutoGenerated: true } OR { title: null }`), so a
  title-first Hub has real data.
- [lib/deliberations/visibility.ts](../lib/deliberations/visibility.ts) —
  `listableDeliberationWhere(userId)` (public + creator + role-holder).
- Design system in [app/globals.css](../app/globals.css): `.cardv2`, `.modalv2`,
  `.btnv2` (+ `--sm/--lg/--ghost/--icon`), `.articlesearchfield`. Reference
  usage: [app/test/ai-epistemic/page.tsx](../app/test/ai-epistemic/page.tsx) and
  [app/(auth)/login/page.tsx](../app/(auth)/login/page.tsx).

## 2. Goals & non-goals

**Goals**
- Title-driven cards and title/tag search.
- Real sorting (recently active, newest, most claims, most contested).
- Working tag facets (replace the `TODO` chip stub).
- Visibility: **public + viewer's own** (keep `listableDeliberationWhere`).
- Eliminate the N+1 stats queries.
- UI cohering with the platform (`cardv2` / `btnv2` / `articlesearchfield`).

**Non-goals (this pass)**
- No schema changes, no migration (`db push` not needed).
- No title-generation work — `autoNameEnrichment` already owns that.
- No pagination/infinite scroll (keep `take: 40`; note the cap in UI).
- No change to the `/deliberation/{id}` detail routes.

## 3. Phase 1 — Functionality (API-first)

All in **`app/api/hub/deliberations/route.ts`** unless noted.

### 3.1 Surface titles, tags, calls
- Extend `select`: add `title`, `tags`, `visibility`, `calls` (description +
  deadline, latest first), keep `hostType` / `hostId`.
- Stop hardcoding nulls. Card title = `title ?? hostTypeLabel(hostType)`.
- Return `tags` and the open-call (if any) instead of `[]` / `null`.

### 3.2 Title + tag search
- Extend the `q` `OR` to include `{ title: { contains: q, mode: "insensitive" } }`
  and `{ tags: { has: q } }`, alongside the existing id/hostId match.

### 3.3 Sorting (`sort` param)
| value | orderBy | meaning |
|---|---|---|
| `active` (default) | `updatedAt desc` | recently touched |
| `newest` | `createdAt desc` | newest first |
| `claims` | by claim count desc | biggest debates |
| `contested` | by openCQs desc | most unresolved |

`claims` / `contested` sort on the aggregates from §3.5 (fetch a candidate page,
aggregate, sort, slice) — documented as approximate over the 40-row window.

### 3.4 Tag facets
- Return a distinct `facets.tags` list (counts) over the listable set so the chip
  picker is data-driven.
- Server-side filter: when `tags` non-empty, add `{ tags: { hasSome: tags } }`.

### 3.5 Kill the N+1
- Replace per-row queries with batched aggregates keyed on the page's ids:
  - `claim.groupBy({ by: ["deliberationId"], where: { deliberationId: { in } }, _count: true })`
  - open CQs: collect claim ids once, `cQStatus.groupBy` (or a single count grouped
    by the claim→deliberation map). Map back to each row.
- Target: **≤ 3 queries** total regardless of page size.

### 3.6 Response contract (stable shape for the UI)
```jsonc
{
  "items": [{
    "id": "...", "title": "…|null", "host": { "type": "...", "id": "..." },
    "tags": ["..."], "visibility": "public",
    "call": { "description": "...", "deadline": "…|null" } | null,
    "stats": { "claims": 12, "openCQs": 3 },
    "updatedAt": "ISO", "createdAt": "ISO"
  }],
  "facets": { "tags": [{ "tag": "...", "count": 4 }] },
  "total": 40
}
```

## 4. Phase 2 — UI coherence

Rebuild **`app/hub/ui/Hub.tsx`** against the design system; preserve the SWR key /
`useBusMutate` live-refresh wiring.

- **Toolbar**: `articlesearchfield` search input; `btnv2--ghost` sort dropdown;
  call filter; tag chips from `facets.tags` (toggle → `tags` param). Keep the SWR
  `params` driving the key.
- **Cards**: `.cardv2`. Title as headline, host-type label as eyebrow, tags as
  small chips, claims / open-CQs as stat pills (reuse the demo page's pill
  idiom), call as an amber line, relative "last active". Actions become
  `btnv2--sm` (Open / Panel / Synthesis).
- **States**: skeleton on load, friendly empty state, "showing first 40" note
  when `total === 40`.
- **A11y**: labelled controls, focusable chips/links.

## 5. Files touched
- `app/api/hub/deliberations/route.ts` — rewrite (Phase 1).
- `app/hub/ui/Hub.tsx` — rewrite (Phase 2).
- Possibly a tiny `hostTypeLabel` helper (inline or `lib/deliberations/`).

## 6. Risks & checks
- `claims`/`contested` sort is window-approximate (40-row cap) — acceptable
  pre-pagination; called out in UI. Revisit if pagination lands.
- Confirm `tags` are actually populated in seed/live data; if sparse, the chip
  picker degrades gracefully (hidden when `facets.tags` empty).
- Keep `listableDeliberationWhere` — do **not** regress to public-only (owner must
  still see their own).
- After edits: `npm run lint`; spot-check the route with a couple of `q` / `sort`
  / `tags` combinations.

## 8. Change record

### 2026-06-30 — Phase 1 (API) complete
`app/api/hub/deliberations/route.ts` rewritten:
- Surfaces `title`, `tags`, `visibility`, and the latest `call` (description +
  deadline); card `label` falls back to a `hostTypeLabel(hostType)` when title is
  null.
- `q` now matches `title` (insensitive) and `tags` (`has`) alongside id/hostId.
- `sort` param: `active` (default, `updatedAt`), `newest` (`createdAt`), `claims`,
  `contested` — the latter two re-rank the 40-row window in memory (approximate, as
  designed).
- Tag facets: `facets.tags` (tag + count) over the listable+`q` set, ignoring the
  `tags` filter so chips never self-empty. Server filter via `tags: { hasSome }`.
- `calls=open` filters to deliberations with at least one call.
- N+1 removed: stats now come from one `claim.findMany` (counts plus the
  claim→deliberation map) and one `cQStatus.groupBy` — 4 queries total
  (deliberations, claims, CQs, facets), flat in page size.
- Response: `{ items, facets: { tags }, total }`. Kept `listableDeliberationWhere`
  (public + viewer's own) per the visibility decision.

Verified: `npx tsc --noEmit -p tsconfig.json` (no errors in route) · `next lint`
clean. Confirmed on live data (titles, tags, host-type fallbacks all render).

### 2026-06-30 — Host-title fallback + title clamp

- `route.ts`: untitled deliberations now fall back to their host's own name —
  `Article.title` / `Stack.name` — before the generic type label. Resolved label
  is `title ?? hostName ?? hostTypeLabel`. Batched per host type (2 extra queries
  only when untitled article/stack rows are present), keyed `${hostType}:${hostId}`.
- `Hub.tsx`: title headline gets `line-clamp-2` (ellipsis after 2 lines) with a
  `title=` tooltip carrying the full text.

### 2026-06-30 — Drop orphan article/stack rows

- `route.ts`: untitled `article` / `library_stack` deliberations whose `hostId`
  resolves to no live host (seed/test rows with synthetic ids like
  `seed:host:…`, `host:giqc27`, `"1"`) are filtered out before stats/items.
  Titled rows and other host types are always kept. Host resolution moved ahead
  of the filter; stats now compute over the post-filter id set.
- Trade-off (accepted): a page can return fewer than the 40-row window — e.g. on
  live data 14 of 40 dropped, 26 shown, all with real titles/host names.
